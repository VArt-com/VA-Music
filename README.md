# Music World

Платформа для независимых музыкантов: регистрация, загрузка треков, бесплатное
прослушивание и скачивание, плейлисты, поиск, кнопка доната на Ko-fi.

Стек: **Next.js 14 (App Router) + Supabase** (база данных, авторизация, хранение
файлов). Хостинг — **Vercel** (бесплатный тариф хватает для старта).

## Что уже готово

- Регистрация / вход по email и паролю
- Загрузка трека (аудиофайл + опциональная обложка, жанр, теги, описание)
- Лента треков на главной с поиском по названию
- Страница трека с плеером и кнопкой скачивания (бесплатно, без ограничений)
- **Удаление собственных треков и видео** — кнопка «Удалить» с подтверждением, видна
  только автору; чистит и файл, и обложку в Storage, и запись в базе
- Профиль артиста со списком его треков и видео
- Публичные плейлисты (создание + просмотр)
- Счётчики прослушиваний и скачиваний
- Кнопка доната на Ko-fi в шапке и подвале сайта
- **Новый тёмный неоновый дизайн** (фиолетово-розовые градиенты, стеклянные карточки,
  неоновая типографика) на всех страницах
- **Глобальный плеер-баннер** внизу экрана с живой анимированной визуализацией частот
  трека (canvas + Web Audio AnalyserNode) — виден на любой странице, пока играет музыка
- **Эквалайзер** (бас/средние/высокие + пресеты: рок, поп, электроника, вокал, бас-буст)
  прямо в плеере
- **Шеринг** — кнопка «Поделиться» на треках и видео: нативный Share на мобильных,
  либо Telegram / WhatsApp / VK / X / Facebook / копирование ссылки
- **Раздел «Видео»** (`/videos`) — отдельная загрузка, лента, просмотр и скачивание
  видеоклипов, устроен так же, как раздел с треками
- **Микшер** (`/mixer`) — две деки с выбором любых треков платформы, кроссфейдер между
  ними и запись получившегося микса прямо в браузере (кнопка «Скачать микс», WebM)
- **Амбиентная аудио-визуализация по бокам** — на широких экранах (от 1280px) слева
  и справа крутятся два одинаковых цветных «prism-орба» (правый — зеркало левого);
  во время воспроизведения они реагируют на настоящие частоты трека через тот же
  `AnalyserNode`, что и эквалайзер (не декоративная CSS-петля), в покое — мягкое
  дыхание. Ничего не перекрывает контент (`pointer-events: none`, только в пустых
  полях по краям)
- **4 языка интерфейса** — русский, English, עברית, العربية. Переключатель языка в
  шапке сайта; выбор сохраняется в cookie на год. Для иврита и арабского сайт
  автоматически переключается в RTL (`dir="rtl"`)

## 1. Создай проект в Supabase

1. Зайди на [supabase.com](https://supabase.com) → New project (бесплатный план).
2. Дождись создания проекта, затем открой **SQL Editor** → New query.
3. Скопируй содержимое файла `supabase/schema.sql` из этого проекта, вставь и
   нажми **Run**. Это создаст все таблицы (включая новую `videos`), политики
   безопасности (RLS), storage-бакеты (`tracks`, `covers`, `avatars`, `videos`) и
   функции для счётчиков.
   > Если у тебя уже был проект в Supabase до появления раздела «Видео» — просто
   > заново вставь и запусти обновлённый `schema.sql`: все команды написаны через
   > `create table if not exists` / `drop policy if exists`, так что скрипт
   > безопасно доустановит только то, чего не хватает (таблицу `videos`, бакет
   > `videos`, RPC-функции для видео и недостающую policy на удаление обложек).
4. Проверь **Storage** в боковом меню — там должны появиться четыре публичных
   бакета: `tracks`, `covers`, `avatars`, `videos`.
5. В **Project Settings → API** скопируй `Project URL` и `anon public` ключ.
6. В **Authentication → URL Configuration** укажи:
   - Site URL: адрес твоего сайта после деплоя (например `https://music-world.vercel.app`)
   - Redirect URLs: `https://music-world.vercel.app/auth/callback` (и то же самое
     для `http://localhost:3000/auth/callback` на время локальной разработки)
7. (Опционально) В **Authentication → Providers → Email** можно отключить
   "Confirm email", если хочешь, чтобы пользователи входили сразу после
   регистрации без подтверждения почты — удобно для тестов.

## 2. Настрой проект локально

```bash
cd music-platform
npm install
cp .env.example .env.local
```

Открой `.env.local` и вставь свои значения:

```
NEXT_PUBLIC_SUPABASE_URL=https://твой-проект.supabase.co
NEXT_PUBLIC_SUPABASE_ANON_KEY=твой-anon-ключ
NEXT_PUBLIC_KOFI_URL=https://ko-fi.com/varthur
NEXT_PUBLIC_WHATSAPP_NUMBER=79991234567
```

> `NEXT_PUBLIC_WHATSAPP_NUMBER` — твой номер в международном формате, только
> цифры, без `+`, пробелов и скобок (например `79991234567`). Если оставить
> пустым/не задавать — кнопка WhatsApp в шапке и подвале просто не покажется.

Запусти локально:

```bash
npm run dev
```

Открой [http://localhost:3000](http://localhost:3000) — сайт должен работать.

## 3. Деплой на Vercel (бесплатно)

1. Залей проект на GitHub (создай репозиторий и запушь код).
2. Зайди на [vercel.com](https://vercel.com) → New Project → выбери репозиторий.
3. В настройках проекта добавь те же переменные окружения, что и в
   `.env.local` (`NEXT_PUBLIC_SUPABASE_URL`, `NEXT_PUBLIC_SUPABASE_ANON_KEY`,
   `NEXT_PUBLIC_KOFI_URL`).
4. Нажми Deploy. После деплоя обнови Site URL и Redirect URLs в Supabase на
   реальный адрес Vercel (или свой домен).

## Важно про авторские права

Так как пользователи сами загружают музыку, а платформа раздаёт её бесплатно —
стоит явно предупредить (в форме загрузки это уже есть текстом), что
загружать можно только:

- собственные оригинальные треки, или
- музыку, на распространение которой есть разрешение правообладателя.

Рекомендуется добавить страницу "Условия использования" и механизм жалоб на
нарушение авторских прав (например, форму или email для DMCA-обращений) перед
тем, как открывать сайт для широкой аудитории.

## Визуализатор, эквалайзер и микшер — как это работает

Плеер, эквалайзер и микшер используют браузерный **Web Audio API**: аудио
пропускается через цепочку `MediaElementSource → фильтры (бас/сред/выс) →
AnalyserNode (для визуализации) → выход`. Для этого браузеру нужно читать
аудиофайл с другого домена (Supabase Storage), поэтому у `<audio>`/`<video>`
элементов выставлен `crossOrigin="anonymous"`. Публичные бакеты Supabase
Storage по умолчанию отдают файлы с корректными CORS-заголовками, так что
дополнительная настройка обычно не нужна — но если визуализатор/эквалайзер не
заработает на проде, в первую очередь проверь CORS в настройках Storage.

Запись микса в `/mixer` делает то же самое через `MediaRecorder`, захватывая
`MediaStreamAudioDestinationNode` с уже сведённым (через кроссфейдер) звуком
обеих дек, и отдаёт результат как файл `.webm` для скачивания — без сервера,
целиком в браузере пользователя.

## Как устроена локализация (RU/EN/HE/AR)

Без изменения URL-структуры (`/track/...`, `/videos`, и т.д. остаются как есть):
текущий язык хранится в cookie `mw_locale`. Серверные страницы читают её через
`getDictionary()` (`src/lib/i18n/server.ts`), клиентские компоненты — через
`useI18n()` из `I18nProvider` (обёрнут вокруг всего приложения в
`src/app/layout.tsx`). Переключатель языка в шапке сайта меняет cookie и
обновляет страницу. Словари лежат в `src/lib/i18n/dictionaries/*.ts` — чтобы
добавить/поправить перевод, редактируй соответствующий файл (`ru.ts`, `en.ts`,
`he.ts`, `ar.ts`), структура типа `Dictionary` в `types.ts` держит их в
синхронизации. Для иврита и арабского `<html dir="rtl">` подключается
автоматически.
